topic: 尽量分离前后端

把分页交给前端去做，后端从数据库中取出数据封装成一个 json 文件，格式如下:
  {
    code: 1,
    message: 'success',
    info: {
      page_size: 6,     // 每一页的留言数量
      total_nums: 10, // 总的留言数量
      data: [     // 二维数组，包含留言信息
        [],
        [],
        ...
      ]
    }
  }
前端 异步请求(ajax)获取该 json 文件，然后进行处理。

小结:
    前后端尽可能的分离的好处也是显而易见的。
    如，分工明确，清楚明了，耦合性低....
